Java中的String、StringBuffer、StringBuilder区别

  • Java

一、 String类

String对象是不可变的。查看JDK文档会发现，String类中每一个看起来会修改String值的方法，实际上都是创建了一个全新的String对象。

1.1 String类内部变量

String类内部拥有以下属性：

private final char value[];    // 存储String对象的字符串
private int hash;                // 当计算过HashCode时，缓存

1.2 String对象的初始化

常用的String对象初始化方法有以下几种：

1.2.1 特殊语法

由于String对象特别常用，所以对String对象进行初始化时，Java提供了一种简化的特殊语法，格式如下：

String s = "abc";

按照标准语法，其格式为：

String s = new String("abc");

但通过标准面向对象的语法来创建String对象，在内存上会创建两个String对象，一个是”abc”对象，存储在常量空间中，一个是使用new关键字为对象s申请的空间。

1.2.2 传递String

由于String对象是不可变的，用一个String对象初始化另外一个String对象没有太大意义，源码中只是将原String对象内部的value数组和hash值拷贝到新的String对象中（value只是拷贝引用）。

public String(String original) {
    this.value = original.value;
    this.hash = original.hash;
}

1.2.3 传递char[]

当传递char[]数组初始化String对象时，String类会将该char[]数组深度拷贝，后续对该数组的修改不会影响到新创建的String对象。

public String(char value[]) {
    this.value = Arrays.copyOf(value, value.length);
}

1.3 String类提供的方法

1.3.1 charAt(int index)方法

该方法首先检查index是否超过了内部字符串数组的大小，如果未超过，则直接方法value[index]。

public char charAt(int index) {
    if ((index < 0) || (index >= value.length)) {
        throw new StringIndexOutOfBoundsException(index);
    }
    return value[index];
}

1.3.2 concat(String str)方法

该方法会首先生成一个str.length+原字符串length大小的数组，将两个字符串数组拷贝进去，再根据该数组创建新一个String对象。

public String concat(String str) {
    int otherLen = str.length();
    if (otherLen == 0) {
        return this;
    }
    int len = value.length;
    char buf[] = Arrays.copyOf(value, len + otherLen);
    str.getChars(buf, len);
    return new String(buf, true);
}

1.3.3 indexOf(String str)或者indexOf(int ch)

在当前String中查找目标串str或者ch开始的第一个位置，前者提供了O(n*m)的查找效率，后者为O(n)。

1.3.4 compareTo(String anotherString)方法

String类实现了Comparable接口，自身带有”可比较”的属性，比如的原则为：原字符串大于要anotherString则返回正数，小于则返回负数，等于则返回0.

public int compareTo(String anotherString) {
    int len1 = value.length;
    int len2 = anotherString.value.length;
    int lim = Math.min(len1, len2);
    char v1[] = value;
    char v2[] = anotherString.value;

    int k = 0;
    while (k < lim) {
        char c1 = v1[k];
        char c2 = v2[k];
        if (c1 != c2) {    // 当两个不等时，则返回c1-c2
            return c1 - c2;     // 如果c1 > c2 则为正数，说明原String大于anotherString，否则c2 > c1返回负数
        }
        k++;
    }
    return len1 - len2;    // 当两个String前缀匹配时，则长度长的获胜
}

二、StringBuffer和StringBuilder

JDK实现中的StringBuffer和StringBuilder都继承自AbstractStringBuilder，这里先讲下AbstractStringBuilder的实现原理，再分析StringBuffer和StringBuilder区别。

1. AbstractStringBuilder实现

我们知道使用StringBuffer等无非就是为了提高Java中字符串连接的效率，如果直接使用”+”来连接字符串的话，为了生存最终的String对象，JVM会创建多个中间对象，因此造成一些需要垃圾回收的中间对象。

针对以上，AbstractStringBuilder类中有以下属性：

char[] value;        // 保存字符串
int count            // 当前char[]数组中使用了多少数量字符

当append的字符串长度超过当前的char数组容量时，则对char数组进行动态扩展，即重新申请一段更大的内存空间，并全部拷贝过去。源码如下：

public AbstractStringBuilder append(String str) {
    if (str == null)
        return appendNull();
    int len = str.length();    
    ensureCapacityInternal(count + len); // 扩展char[]数组大小，保证容量可容纳
    str.getChars(0, len, value, count); // 将str中的字符串拷贝到value后面的空间中
    count += len;
    return this;
}

private void ensureCapacityInternal(int minimumCapacity) {
    // overflow-conscious code
    if (minimumCapacity - value.length > 0) // 是否需要扩展
        expandCapacity(minimumCapacity);
}

void expandCapacity(int minimumCapacity) {
    int newCapacity = value.length * 2 + 2; // 扩展两倍
    if (newCapacity - minimumCapacity < 0)
        newCapacity = minimumCapacity;
    if (newCapacity < 0) {
        if (minimumCapacity < 0) // overflow
            throw new OutOfMemoryError();
        newCapacity = Integer.MAX_VALUE;
    }
    value = Arrays.copyOf(value, newCapacity); // 扩展后拷贝
}

2. StringBuilder与StringBuffer的区别

在StringBuffer源码中，增删改查等方法前面都带有synchronized关键词，其保证了线程安全性。主要针对一个线程在查询字符串时，其他线程不允许修改，否则会造成问题。

而StringBuilder没有线程安全性的考虑，其主要原因在于一般的字符串操作都是在单个线程中进行，所以该类中字符串操作会快一点。
